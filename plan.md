# 実装・運用プラン

1. DB と通知の堅牢化
   - `posted_at` と `slack_ts` を `articles` に保存して重複防止と追跡を可能にする（完了）
   - Slack 投稿に失敗した場合は `posted_at` をセットしない（保持して再試行できるようにする）

2. LLM 出力の堅牢化
   - モデルが空や非JSONを返した場合、記事本文の先頭を要約の代替として使う（実装済）
   - プロンプトを日本語・厳密JSON要求に調整（実装済）

3. テスト
   - ユニットテストを追加して検証（既存のテストを更新・追加済み）

4. デプロイ/実行手順
   - `.env` を確認してから `docker compose build` → `docker compose up -d` → `docker compose logs -f collector`
   - 手動トリガーや再試行フローの確認手順をREADMEに記載

5. 次の作業候補
   - Slack投稿の返信・削除時の追跡（`slack_ts` を使った処理）
   - 投稿失敗のリトライキュー実装
   - LLMへのプロンプト最適化（温度調整、分割処理）
